## Overview

`SlideNotesTTS` is a Python CLI tool that converts presentation Markdown files generated by [markitdown](https://github.com/microsoft/markitdown) into MP3 audio files using Microsoft's [edge-tts](https://github.com/rany2/edge-tts) text-to-speech engine. It supports converting slide text, speaker notes, or both as separate audio files per slide, with interactive voice selection including language and region filtering, a voice test before committing, and a saved voice preference for faster repeat runs.

## Background

During the first year of a doctoral programme it became clear that reading lecture slides, articles, and presentations at a desk was using up time that could otherwise be spent studying. The programme is demanding and covers research methodology, strategic management, innovation, and sustainability across three years, a lot of material to absorb. Converting presentation files to audio made it possible to go through course content during commutes, exercise, and other time away from a screen.

`SlideNotesTTS` was built to automate that conversion. Given that `markitdown` already handles the extraction of structured text from PowerPoint files, the remaining problem was turning that output into clean, natural-sounding audio in a single command, across an entire deck, with control over what gets converted and in which voice.

## Demo

The `demo/` directory contains three sample presentations in American English, British English, and Croatian, along with pre-generated MP3 output. The audio files are a quick way to hear what the tool produces before running it on your own materials.

## Usage

```text
python3 SlideNotesTTS.py <file> [file ...] [options]
```

### Arguments

| Argument | Description |
| --- | --- |
| `file` | Path to one or more markitdown-generated `.md` files. |

### Options

| Option | Description |
| --- | --- |
| `--voice <name>` | Use this voice and skip interactive selection. The name is validated against available voices at startup. Example: `en-US-JennyNeural`. |
| `--lang <code>` | Pre-filter voices by language code (e.g. `en`, `hr`, `de`). |
| `--region <code>` | Pre-filter voices by region code (e.g. `US`, `GB`, `AU`). |
| `--mode <1\|2\|3>` | Content mode: `1` = slide text only, `2` = speaker notes only, `3` = both. |
| `--output <dir>` | Write all output to the specified directory. For multiple input files, each gets a subdirectory named after the source file inside `<dir>`. |
| `--jobs <N>` | Number of parallel generation workers (default: `1`). Use `--jobs 4` to generate up to four audio files simultaneously. |
| `--timestamp` | Append a run timestamp to output filenames. Useful for versioned or archived runs. |
| `--dry-run` | Parse files and print what would be generated without creating any audio. Useful for verifying mode and voice before committing a large batch. |
| `--force` | Overwrite MP3 files that already exist. By default, files present in the output directory are skipped. |
| `--list-voices` | List available voices (filtered by `--lang` / `--region` if provided) and exit. |
| `--version` | Print the version number and exit. |
| `--retry` | Retry failed files once before exiting. |
| `--reset` | Ignore saved voice preference and force re-selection. |

## Voice selection

When no voice is saved, after `--reset`, or when the saved voice is no longer available, the tool presents an interactive voice selection flow:

1. Displays all available language codes.
2. Prompts for a two-letter language code (e.g. `en`, `hr`).
3. Displays available regions for that language and prompts to filter or use all.
4. Displays the matching voices and prompts for a selection by number.
5. Optionally generates and plays a short audio sample for confirmation.

The selected voice is saved to `~/.slidenotes_tts` and offered as the default on subsequent runs. Use `--reset` to force re-selection, or `--voice` to bypass the flow entirely.

## Examples

Interactive run with a single file:

```sh
python3 SlideNotesTTS.py lecture_01.md
```

Pass voice and mode directly, skipping all prompts:

```sh
python3 SlideNotesTTS.py lecture_01.md --voice en-US-JennyNeural --mode 2
```

Process all lectures in a batch, notes only, with retry on failure:

```sh
python3 SlideNotesTTS.py lecture_01.md lecture_02.md lecture_03.md --mode 2 --retry
```

Process a large deck faster using 4 parallel workers:

```sh
python3 SlideNotesTTS.py lecture_01.md --jobs 4
```

Write all output to a specific directory:

```sh
python3 SlideNotesTTS.py lecture_01.md lecture_02.md --output ~/audio/lectures
```

Preview what would be generated without creating any files:

```sh
python3 SlideNotesTTS.py lecture_01.md --dry-run --voice en-US-JennyNeural --mode 2
```

Regenerate a deck where the audio already exists:

```sh
python3 SlideNotesTTS.py lecture_01.md --force
```

Keep a timestamped archive of each generation run:

```sh
python3 SlideNotesTTS.py lecture_01.md --timestamp
```

Pre-filter to British English voices only:

```sh
python3 SlideNotesTTS.py lecture_01.md --lang en --region GB
```

List all Croatian voices:

```sh
python3 SlideNotesTTS.py --list-voices --lang hr
```

## Output

For each processed file, a directory named `<presentation>_audio/` is created alongside the source file. When `--output` is specified, output goes to the given directory instead (one subdirectory per file when processing multiple files). Output files are named using a zero-padded slide number and the selected voice:

```text
lecture_01_audio/
    slide_01_content_en-US-JennyNeural.mp3
    slide_01_notes_en-US-JennyNeural.mp3
    slide_02_content_en-US-JennyNeural.mp3
    slide_02_notes_en-US-JennyNeural.mp3
    ...
```

Because filenames are stable across runs, re-running with the same voice skips files that already exist. Different voices produce different filenames and coexist in the same directory. Pass `--timestamp` to append a run timestamp for versioned output:

```text
    slide_01_notes_en-US-JennyNeural_20260619-143022.mp3
```

A per-file summary is printed on completion, and a combined total summary is shown when processing multiple files.

## Files

1. `SlideNotesTTS.py`: The main script.
2. `~/.slidenotes_tts`: Config file created automatically to store the last used voice.
3. `<presentation>_audio/`: Output directory created alongside the source file, containing the generated MP3 files.
4. `demo/`: Sample presentation files and pre-generated audio output in three languages.

## Dependencies

- Python 3.8 or later.
- [edge-tts](https://github.com/rany2/edge-tts): Microsoft Edge TTS engine.
- [rich](https://github.com/Textualize/rich): Terminal UI.

Install dependencies:

```sh
pip3 install edge-tts rich
```

The source file must be a Markdown file produced by `markitdown` from a PowerPoint presentation. The tool expects `<!-- Slide number: N -->` markers and optional `### Notes:` sections within each slide block.

## Notes

- The source `.md` file must have been generated by `markitdown` from a PowerPoint file. Plain Markdown files without `<!-- Slide number: N -->` markers will not be parsed.
- Slides or notes that are empty after text cleaning are skipped and reported. Empty text is the result of slides that contain only images, diagrams, or unsupported formatting.
- MP3 files that already exist in the output directory are skipped and reported. Use `--force` to overwrite them, or `--dry-run` to preview which files would be affected before committing.
- `edge-tts` requires an active internet connection as it routes requests through Microsoft's servers.
- Voice availability varies by language and region. Use `--list-voices` to browse what is available.
- Exit codes: `0` = success, `1` = at least one file failed, `2` = nothing generated (all files already existed or all slides were empty after cleaning), `130` = keyboard interrupt.
